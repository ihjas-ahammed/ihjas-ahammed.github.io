Day 6: Data Preprocessing Techniques

Understand common methods for cleaning, integrating, transforming, and reducing data.

Topics (from mod3.md)

  • Binning (smoothing local values), Regression (fit curve), Clustering (detect outliers), Combined human/computer inspection. Brief concept of each.
  • Statistical (Z-score > threshold), IQR method (outside Q1-1.5*IQR or Q3+1.5*IQR), Isolation Forest, DBSCAN (density-based clustering). Understand *what* they do, not deep algorithms.
  • Combining data from multiple sources. Challenges: Entity identification, Redundancy, Data value conflicts, Schema integration, Duplication.
  • Changing data format or structure (e.g., scaling, aggregation).
  • Purpose: Put features on a similar scale for distance-based algorithms (KNN, K-Means, PCA). Z-Score (StandardScaler: mean=0, std=1), Min-Max (MinMaxScaler: range [0,1]), Robust Scaling (uses IQR, robust to outliers), Normalizer (scales samples/rows to unit norm). Understand goal & basic idea.
  • List & brief concept: Smoothing (remove noise), Aggregation (summarize), Discretization (numeric to categorical bins), Attribute/Feature construction, Generalization (concept hierarchies).
  • Obtain reduced representation while maintaining integrity. Strategies: Dimensionality Reduction (features), Numerosity Reduction (data volume), Data Compression.
  • Concept: Create new uncorrelated features (Principal Components) that capture maximum variance. Reduce number of features while retaining most information.
  • Pandas: `.replace()`, `.dropna()`, `.fillna()`, `.drop_duplicates()`. Scikit-learn: `SimpleImputer`. Z-score removal logic. Understand *what* the code achieves.

Focus

Key techniques for fixing data issues and preparing it for analysis/modeling. Understand the purpose of scaling and PCA.

Study Tips (1 Hour Session)

  • Focused Study (40-45 mins): Understand the *purpose* and basic idea of each technique (cleaning, scaling, reduction).

Day 7: Machine Learning Introduction

Get a high-level overview of Machine Learning concepts, types, evaluation, and key algorithms mentioned.

Topics (from Unit 4 .md)

  • Definition (Learn from data without explicit programming), Importance (automation, insights).
  • Supervised (labeled data), Unsupervised (unlabeled data), Semi-supervised (mix), Reinforcement (rewards/penalties). Know the key difference based on data/feedback.
  • Task types: Classification (predict category, e.g., spam/not spam) vs. Regression (predict continuous value, e.g., house price).
  • Task types: Clustering (group similar data points, e.g., customer segments) vs. Dimensionality Reduction (reduce features, e.g., PCA).
  • Key Components: Agent, Environment, State, Action, Reward. Learning through trial and error.
  • Why? Train model on one part of data, evaluate its performance on unseen data (test set) to estimate generalization.
  • Bias: Error from wrong assumptions (Underfitting - too simple). Variance: Error from sensitivity to small fluctuations in training data (Overfitting - too complex). Goal is low bias & low variance.
  • Classification: Accuracy (overall correct), Confusion Matrix (TP/TN/FP/FN), Precision (TP / (TP+FP) - correctness of positive predictions), Recall (TP / (TP+FN) - sensitivity, finds all positives), F1-Score (harmonic mean of Precision/Recall). Regression: MAE, MSE, RMSE (error magnitudes), R² (variance explained). What they measure.
  • Concept: Classify/predict based on majority vote/average of 'K' nearest neighbors in feature space (using distance metric). Simple, instance-based.
  • Concept: Unsupervised. Partition data into 'K' clusters. Iteratively assign points to nearest centroid and update centroid position.
  • Concept: Probabilistic classifier based on Bayes' Theorem with 'naive' assumption of feature independence. Fast, good for text.
  • Concept: Matrix factorization technique. Used in dimensionality reduction, recommendations.

Focus

Understand the ML landscape, the purpose of different learning types, the crucial concepts of train/test split and bias/variance, and the basic idea behind the mentioned algorithms (KNN, K-Means).
